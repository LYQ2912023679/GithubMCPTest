Python贪吃蛇游戏需求文档 (PRD)
1. 引言
1.1 编写目的
本文档旨在明确“Python贪吃蛇游戏”的功能需求、非功能需求及技术栈，为开发人员提供清晰的开发指南，确保最终交付的游戏产品符合预期标准。

1.2 项目范围
本项目是一个基于Python开发的单机2D桌面小游戏。玩家通过控制蛇的移动来吃食物，使蛇变长并获得分数。游戏包含基本的开始、运行、暂停和结束状态。

2. 总体描述
2.1 用户特征
目标用户为所有年龄段的PC端用户，具备基础的键盘操作能力，寻求简单、易上手的休闲游戏体验。

2.2 运行环境
操作系统：Windows / macOS / Linux
Python版本：Python 3.6 及以上
依赖库：pygame（推荐）或 tkinter / curses
3. 功能需求
3.1 游戏初始化
游戏窗口：创建固定大小的游戏窗口（例如 600x600 像素），设置窗口标题为“贪吃蛇游戏”。
网格系统：游戏画面基于隐形的网格系统（例如 20x20 像素一个格子），蛇和食物的移动与生成均对齐网格。
初始状态：游戏启动时显示“开始界面”，包含游戏名称和“按任意键开始”的提示。
3.2 蛇的移动与控制
初始属性：蛇初始长度为3节，位于屏幕中央，初始移动方向为向右。
方向控制：
支持方向键（↑ ↓ ← →）和 WASD 键控制方向。
防反向规则：蛇不能直接向当前移动的反方向移动（例如向右移动时不能直接按左键）。
移动机制：蛇以固定的时间间隔（如每0.15秒）移动一格。
3.3 食物生成
随机生成：在游戏网格内随机生成一个食物点。
避障规则：食物生成的位置不能与蛇身体的任何一节重叠。
外观：食物需有明显的颜色区分（如红色），与背景和蛇身形成对比。
3.4 碰撞检测
吃到食物：当蛇头与食物坐标重合时：
蛇身长度增加一节（蛇尾不消失）。
当前分数增加（如+10分）。
游戏速度略微提升（可选功能，增加难度）。
在新位置生成下一个食物。
撞墙判定：当蛇头超出游戏窗口边界时，触发游戏结束。
撞自身判定：当蛇头与蛇身任何一节坐标重合时，触发游戏结束。
3.5 计分系统
当前分数：在游戏界面顶部或侧边实时显示当前得分。
最高分：游戏结束时，将当前分数与历史最高分对比并更新。最高分需持久化存储（可写入本地 .txt 或 .json 文件）。
3.6 游戏状态管理
开始状态：显示游戏标题和操作说明。
运行状态：正常进行游戏逻辑。
暂停状态：按下 空格键 暂停游戏，画面显示“已暂停”，再次按下恢复。
结束状态：显示“游戏结束”、本局得分、历史最高分，以及“按 R 键重新开始 / 按 ESC 键退出”的提示。
4. 非功能需求
4.1 性能需求
帧率控制：游戏主循环需保持稳定（如60 FPS用于渲染，但蛇的移动逻辑通过定时器或帧数计数器控制，避免移动过快）。
资源占用：CPU和内存占用应极低，确保在普通办公电脑上流畅运行。
4.2 用户界面 (UI/UX)
色彩搭配：背景建议使用深色（如黑色或深蓝色），蛇身使用亮色（如绿色或黄色），食物使用对比色（如红色），确保视觉清晰。
响应反馈：按键操作需即时响应，无明显延迟。
4.3 代码结构
遵循模块化设计，建议将游戏分为以下模块：
main.py：主循环与状态控制。
snake.py：蛇的属性与行为逻辑。
food.py：食物的生成逻辑。
settings.py：常量配置（颜色、尺寸、速度等）。
5. 控制说明
按键	功能
↑ / W	蛇向上移动
↓ / S	蛇向下移动
← / A	蛇向左移动
→ / D	蛇向右移动
空格键	暂停 / 继续
R	游戏结束后重新开始
ESC	退出游戏
6. 交付物
完整的Python源代码（包含注释）。
requirements.txt 依赖说明文件。
历史最高分记录文件（如 highscore.json）。
（可选）打包后的可执行文件（.exe）。
